## Systemic finding

1. **KR brands remain invisible to Tier-2 directories** — getdesign.md and styles.refero.design returned **0/10** entries (getdesign.md explicitly: "No designs found"). Consistent with every prior KR batch. All tokens rest entirely on Tier-1 live inspect; the KR regional rule (≥2 brand-owned Tier-1 surfaces) carried every entry.
2. **Banking/payment consumer apps are login-gated; the public web surface is the corporate/marketing shell.** Naver Pay's `new.pay.naver.com` is fully auth-gated → tokens came from the merchant-center pre-login landing + the official **Naver Pay Developers design guide** (a genuine brand-owned DS surface). KB Pay's `kbpay.kbcard.com` 404s → the live surface is `card.kbcard.com`. The in-app token sets for all four fintech entries sit behind native shells and are an honest UPDATE target.
3. **Two design registers per legacy financial brand.** Hana (retail `kebhana.com` #00a39f / NotoSans vs. group `hanafn.com` #009178 / Pretendard) and Shinhan (classic internet-banking #2967B2 / Spoqa vs. modern group+SOL #005DF9 / OneShinhan + Pretendard) each ship a documented dual-era system — captured as intentional layering, not conflict.
4. **Automotive = monochrome + zero-radius + shadowless.** Both Hyundai (#002c5f, 0px radius, HyundaiSans) and Kia (#05141f, 0px, Kia Signature) push all visual depth onto vehicle photography. Samsung's commerce surface is similarly monochrome (pure-black #000000 pill CTA, no brand hue on web — One UI blue #0381fe governs only the Galaxy OS layer).

## Known shared limitation

Korea's most-used financial/payment products live almost entirely inside **login-gated native apps**; the publicly inspectable web is the corporate/marketing shell plus (where they exist) developer/brand DS surfaces. This caps §4 component depth for naverpay/kbpay at the pre-login + developer-guide layer. Process improvement for future KR fintech batches: prioritize the brand's **developer/design-guide subdomain** (e.g. `developers.pay.naver.com/design`) as the second surface from the start — it is brand-owned, unauthenticated, and token-rich, and outperforms PDP/`/about` deep-links that SPA-redirect to root.
