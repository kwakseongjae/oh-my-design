# Reference batch audit — 2026-06-26 · KR×30

30 new Korean company references added (catalog 326 → 356). KR refs 119 → 149.
Pipeline: 탐색(8-agent sector + DS-first discovery) → 선발(dedupe vs 326, category-balanced 30 + user confirm) → 수집(per-brand CREATE subagents, waves of 5) → 검수(central verify-reference + catalog-integrity) → 배포(sync-catalog).

## How the 30 were researched

- **탐색**: a discovery workflow fanned 8 agents across Korean sectors (fintech/banking/crypto, commerce/fashion/beauty, gaming/content/media, SaaS/devtools/productivity, edtech/HR, health/pet/mobility/travel/proptech, AI/data, + a sector-agnostic DS-first sweep). 101 raw → 89 unique candidates after deduping against the full 326-id catalog. Each candidate's brand-owned design/eng surfaces were HEAD/WebFetch-verified live before it could be proposed.
- **선발**: a synthesis pass picked 30 best-first under (1) KR-proof-gate buildability (≥2 brand-owned surfaces), (2) design-system maturity, (3) portfolio balance toward under-covered categories, (4) recognition. Two LLM-drift duplicates were caught at the orchestrator dedup step (`wadiz` already fintech; `yeogi` == existing `yeogiotte`/여기어때) and replaced with clean alternates (`queenit`, `weverse`). `zigbang`/`zigzag` alternates were also already in-catalog and excluded.
- **수집**: one subagent per brand ran the `omd:add-reference` CREATE pipeline from `batch-instructions.md` — Tier 1 live inspect (global Playwright `getComputedStyle` over homepage + ≥1 brand-owned surface), Tier 2 attempt (getdesign.md + refero), DESIGN.md + .verification.md authored to the machine gates, self-verified to `verify-reference.mjs` all-green. Run in 6 sequential waves of ~5 (after an initial 16-wide burst tripped a transient server-side rate limit).
- **검수**: every id re-checked centrally with `verify-reference.mjs` (30/30 = 27/27 gates) and the full vitest catalog-integrity suite (16 files / 594 tests green). No stray repo artifacts; every ref dir holds exactly DESIGN.md + .verification.md.

## Systemic finding

Korean brands remain **empty in both Tier-2 catalogs** (getdesign.md, styles.refero.design) — confirmed for all 30. Every reference therefore rests entirely on Tier-1 brand-owned proof, exactly as `spec/regional-sources.yaml` anticipates. Two recurring authoring realities this batch:
1. **Modern KR product sites emit `lab()`/`oklab()`/`oklch()` colors** (airbridge, moreh, and others on Tailwind v4). Agents resolved these to `#rrggbb` in-page (canvas `getImageData`) and documented the conversion in the Proof block — otherwise token↔prose grounding would fail.
2. **Two-surface token systems are common and legitimate** (kyobobook portal vs storefront, elice corporate vs 엘카데미 LXP, cafe24 marketing vs merchant app, portone marketing vs docs, neosapience corporate vs Typecast). These were recorded as intentional splits in the conflict matrix, not as unresolved conflicts.

## Known shared limitation

- All Medium grades trace to one cause: **marketing-site-only or app-first surfaces** where the public web exposes a thin component vocabulary. These are honest caps, not defects — each still passes the proof gate. The fix is a later `omd:add-reference --mode update` / `omd:component-harvest` pass against product/app surfaces, not a re-CREATE.
- `tone_keywords` in the new fingerprints are heuristic (`[fingerprints] … refine keywords by hand if needed`) and lean generic ("bold/warm/dark/dense"). A hand-refinement pass would improve `/curation` matching quality.
- Process improvement carried forward: **cap build concurrency at ~5** for KR batches — the 16-wide first wave hit a transient server-side rate limit (not a usage limit); waves of 5 ran clean.
