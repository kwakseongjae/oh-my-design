## How the 10 were researched

Candidate set was curated + adversarially vetted upstream (HQ-country + design-value verification; see `research/2026-06-17_reference-candidates-kr-us-tw-jp.md`). Each brand was then built by an isolated subagent following `.claude/skills/omd-add-reference/batch-instructions.md`: read SKILL + spec/verification-pipeline + components-schema + regional-sources + stripe/finda exemplars → live-inspect homepage + ≥1 more brand-owned surface via global Playwright `getComputedStyle` → Tier-2 probe (getdesign.md + refero) → write `DESIGN.md` + `.verification.md` → self-verify to green. Two builds (stibee, pinkfong) hit a transport-layer socket error mid-stream and were re-run cleanly.

## Systemic finding

1. **KR brands remain invisible to Tier-2 directories** — getdesign.md and styles.refero.design returned **0/10** entries. Consistent with every prior KR batch (`2026-05-13/14/15`). All tokens rest entirely on Tier-1 live inspect; KR regional rule (≥2 brand-owned Tier-1) carried every entry.
2. **KR corporate/marketing surfaces are increasingly SPA shells.** Nominal `/brand` · `/about` routes and e-commerce PDP deep-links frequently redirect back to the homepage root (NHN `/brand`, IICOMBINED PDP, Stibee `/about`). Second-surface strategy had to use **locale variants** (nhn.com/en-US) or **service subpaths** (tmapmobility `/service/drive/navigation`, `/people/about`; spoqa.github.io typeface site) instead.
3. **Shadowless is the KR default.** 9/10 ship `box-shadow: none` as the dominant chrome; depth comes from tonal contrast + hairlines. Matches the Toss/Bunjang/Dabang discipline noted in prior audits.

## Known shared limitation

App-first brands (zepeto, tmap consumer app, wrtn in-app) expose only their marketing/web shells publicly, so deep component states (dialog/toast/in-app forms) could not be live-inspected. **Process improvement for batch 2+**: for app-first brands attempt mobile-viewport emulation + capture any logged-out app web shell before falling back to editorial extrapolation. Also add a **pre-sync `tokens.colors` hex-format lint** to the subagent self-check — `verify-reference.mjs` grounds colors but does not reject `rgba()`/`hsl()` in the colors map, which only `build-registry` catches (shiftup tripped this).
