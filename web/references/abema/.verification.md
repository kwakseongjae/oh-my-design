## Outage circumstance (method note — honesty)

On the inspect date (2026-06-10, evening JST) ABEMA had a publicly reported large-scale outage starting ~18:16 JST (help.abema.tv announcement 「【お詫び】ただいまABEMAにおいて不安定な状況が発生しています」; covered by nlab.itmedia.co.jp). abema.tv served its dark maintenance/SorryPage shell on all routes (`メンテナンス中 | ABEMA`), and the current asset bundles 404'd at origin. Therefore:

- Live inspect was performed on the outage shell (which runs the brand's real token CSS — values matched the registry tokens exactly: #000 canvas, #212121 surfaces, #ddaa00 links, 4px radius, CopyRight font stack).
- The full app token/component CSS was retrieved from the brand's own shipped artifact `abema.tv/assets/registry.1bbd6d267a32e228541e6.css` via the web.archive.org capture of 2025-12-31 (snapshot of the then-live homepage HTML referenced this exact bundle). All §4 component values and the tokens: block are quoted from this file, not invented.
- times.abema.tv (brand-owned, unaffected) was live-inspected as the second surface.
- abema.tv is additionally geo-sensitive for playback outside Japan; the catalog UI could not be re-rendered from this egress during the outage window.
