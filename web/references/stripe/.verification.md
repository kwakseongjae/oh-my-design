### Major finding: TWO parallel button systems

Stripe runs two distinct design systems on parallel surfaces, similar to Apple's marketing-pill vs commerce-compact split:

#### System A — HDS (Honeycomb Design System) — `stripe.com/` home
- **Primary** (`.hds-button.hds-button--primary`)
  - bg `rgb(83, 58, 253)` = `#533afd` (Deep Violet)
  - color `#ffffff`
  - radius **4px** (sharp)
  - padding 15.5px 24px 16.5px (asymmetric vertical)
  - 48px height
  - 16px / weight 400 / sohne-var
  - "Get started", "Stripe for enterprises", "Stripe for startups"
- **Header CTA** (`.hds-button.navigation-cta-button`)
  - bg `#533afd` / color `#fff` / 4px / 11.5×20 / 40px / 14px·400
  - "Contact sales" header
- **Sign in (link-style)** (`.hds-button.navigation-cta-button` link variant)
  - bg transparent / color `#533afd` / 4px / 10.5×20 / 40px
- **OAuth Outline** (`.hero-section__button`)
  - bg `rgba(255, 255, 255, 0.65)` / color `#533afd` / 4px / 14.5×24 / 48px
  - "Sign up with Google"
- **Inverse Primary** (`.sessions-section__cta`)
  - bg `#fff` / color `#533afd` / 4px / 15.5×24 / 48px
  - "Watch on demand" — used on dark sections
- **Tinted Secondary** (`.customer-story-button`)
  - bg `#e2e4ff` (lavender 100) / color `#533afd` / 4px / 0 padding
  - "Read the story" — story carousel
- **Carousel ghost** (`.hds-ui-button hds-ui-button--quiet`)
  - bg transparent / `rgba(16,16,16,0.3)` / 4px — prev arrow when at start
  - bg `#e8e9ff` / `#000` / 4px — next arrow active

#### System B — Sessions / `.CtaButton` — `stripe.com/payments`
- **Primary** (`.CtaButton variant--Button`)
  - bg `rgb(153, 102, 255)` = **`#9966ff`** (lighter lavender, NOT `#533afd`)
  - color `#ffffff`
  - radius **16.5px** (pill-like, much rounder than HDS 4px)
  - padding 3px 12px 6px 16px (asymmetric)
  - 33px height
  - 15px / weight **425** / sohne-var (note: weight 425, between 400 and 500)
  - "Start accepting payments", "Start now"
- **Link** (`.CtaButton variant--Link`)
  - bg transparent / `#9966ff` / 16.5px / 33px / 15px·425
  - "Try the demo", "Explore full page", "Read the story"
- **Carousel arrows** (`.FullWidthFeatureCarouselNav__button`)
  - bg transparent / `#000` / 14px / 1×6 / 28px

#### Conflict: HDS `#533afd` vs Sessions `#9966ff`

These are **different colors on different surfaces**, not a single brand color with rendering variance:
- `#533afd` (Deep Violet, more saturated) — the iconic Stripe color since founding, used on stripe.com, blog, customer stories
- `#9966ff` (lighter lavender) — newer, used on `/payments` and product-detail pages (probably 2024-2025 redesign)

**Resolution:** Both retained as parallel systems. `#533afd` is still THE Stripe brand color (header, primary marketing); `#9966ff` is the Sessions/Payments product surface accent. Document both in §4 as separate variant categories.
