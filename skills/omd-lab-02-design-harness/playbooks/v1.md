## Agent prompts source

The 8 prompts live at `.claude/agents/omd-*.md` (this repo) and `.codex/agents/omd-*.toml` (Codex mirror). v1 ships them verbatim.

## Eval rubric (jury)

LLM jury rates against 4 dimensions (0-5 each, no compound score):

| Dimension | Rubric criteria |
|---|---|
| Voice fidelity | Forbidden phrases used: 0. Microcopy matches §10 register. |
| State completeness | All 5 states (empty/loading/error/success/skeleton) present per screen. |
| Principle adherence | No §12 Principle violations in wireframes/components. |
| IA discipline | Every screen has entry, exit, error path. |

Pairwise vs previous iteration, both orderings, position bias mitigated.

## Persona pool (default)

For Korean projects:
- 1 from DESIGN.md §13 (user-defined; if missing, swap to nemotron 1×)
- adversarial-impatient (3s budget)
- adversarial-low-literacy
- adversarial-brand-skeptic

For English projects: same skeleton, persona prompts in English.

## Success criteria for v1

A v1 run is "successful" if:
1. Reaches Phase 9 handoff without crash or infinite loop.
2. All 3 mandatory user checkpoints honored (no auto-skip).
3. handoff/v0.zip is non-empty and contains brief + DESIGN.md + wireframes + components/manifest.json + assets/.
4. eval/deterministic.json verdict = "pass".
5. ≥ 2/4 personas reach task_success = true.
6. Total tokens stay under ANY single-day Claude Pro / ChatGPT Plus quota for an active user (informally tracked, not enforced).

## Known v1 limitations (will guide v2-v3 hypotheses)

- Cross-family jury graceful-degrades to single-family if user lacks GPT/Gemini keys → v2 candidate: enforce 2-family minimum or substitute with prompt-diversity ensemble.
- Asset sourcing is API-key-gated (Unsplash / Pexels APIs); without keys, falls back to WebFetch on public pages → v2 candidate: ship with default API keys via package.json env-template, or bundle a public-domain seed pack.
- Persona walkthrough simulation mode (no Playwright) loses motion/click realism → v2 candidate: bundle a static-HTML render layer so simulation has DOM to walk.
- Korean persona pool requires user opt-in (license) → v2 candidate: pre-stage opt-in at first run instead of mid-flow.

## Instrumentation (what v1 measures, mandatory for compare/)

Every v1 run must emit `postmortem.md` with:

```markdown
- iterations: <N>
- total_tokens_estimated: <approx based on agent log>
- persona_abandon_rate: <abandons / total personas>
- deterministic_pass_rate: <passes / total checks>
- jury_score_normalized: <avg rubric score / 5>
- time_to_handoff_min: <ISO end - ISO start, in minutes>
- user_satisfied: yes | partial | no | unknown
- user_reject_phases: [<phases user pushed back on>]
- agent_token_breakdown:
    omd-master: ~Xk
    omd-ux-researcher (×3): ~Xk
    ...
- failure_modes_hit: [<list of named failure modes>]
```

`omd lab compare` reads these to build metrics.json.

## Scope freeze

v1 does NOT include:
- Pixel rendering of components (delegated to v0/Cursor/Subframe)
- A11y violations beyond axe-core + lighthouse (no manual heuristic walk)
- Live A/B testing with real users (synthetic personas only)
- Automatic API key acquisition (user provides)

These are explicit scope decisions — moving them in is a v2+ change, not a v1 bug.
