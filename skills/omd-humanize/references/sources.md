# Research sources and reuse boundary

OmD의 규칙 문장은 아래 프로젝트를 그대로 복제하지 않고, 제품 카피·다국어 UI에 맞게 다시 정의했다.

- [im-not-ai](https://github.com/epoko77-ai/im-not-ai) — 한국어 번역투·피동·형식명사·리듬 taxonomy, 의미 보존과 국소 수정 원칙. MIT.
- [Humanizer](https://github.com/blader/humanizer) — 단일 표현보다 pattern cluster를 보고, 의미를 유지한 채 다시 쓰는 원칙. MIT.
- [stop-slop](https://github.com/hardikpandya/stop-slop) — 군더더기, 기계적 대조, 과장, 균일한 리듬을 줄이는 편집 관점. MIT.
- [Microsoft language style guides](https://learn.microsoft.com/globalization/reference/microsoft-style-guides) — 한국어·일본어·간체·번체를 서로 다른 locale guide로 취급.
- [Mozilla localization style guides](https://mozilla-l10n.github.io/styleguides/) — locale별 용어·문장부호·제품 voice 참고.

실질적인 코드·규칙 문장을 가져올 때는 원 프로젝트의 라이선스와 저작권 고지를 함께 보존한다. 이 스킬은 출처 판별기나 탐지 우회 도구가 아니다.
