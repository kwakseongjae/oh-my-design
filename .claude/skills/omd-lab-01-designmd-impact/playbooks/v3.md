# V3 Playbook — Description → DESIGN.md → UI

## 목적
실제 omd:add-reference 흐름 모방. 자연어 설명을 DESIGN.md로 자동 변환한 뒤 그걸로 UI 생성.

## 단계

### 1. step1-description.md 작성
toss-describer 서브에이전트(Explore)에게 `shared/reference-toss/pinterest/` 8개 핀과
`shared/reference-toss/analysis.md`를 보고 자연어 설명 작성 요청.

```
prompt 요약: "토스 핀 8장을 봤다고 가정하고 디자인 시스템을 자연어로 설명해줘.
색·타이포·레이아웃·컴포넌트·톤·이미저리·모션·철학 순서로.
실제 사용자가 add-reference에 붙여넣을 만한 자연스러운 한국어 문장으로."
```

→ `v3-describe-to-design/step1-description.md`

### 2. step2-DESIGN.md 자동 생성
design-md-author 서브에이전트(sc:sc-technical-writer)에게 step1 → DESIGN.md 변환 요청.
"omd-add-reference 스킬이 하는 일을 흉내내라"고 명시.

→ `v3-describe-to-design/step2-DESIGN.md` (V2와 같은 9+3섹션 구조)

### 3. ui-builder 호출
```
context: shared/base-prompt.md + step2-DESIGN.md
output: v3-describe-to-design/output/
```

### 4. chatlog
`chatlog/v3-session.md`에 step1·step2·UI 빌드 3턴을 모두 박제.
