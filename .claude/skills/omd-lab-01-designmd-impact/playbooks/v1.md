# V1 Playbook — DESIGN.md 없이

## 목적
context로 base-prompt만 전달했을 때 LLM이 만드는 "토스 스타일"의 베이스라인 측정.

## 단계

### 1. ui-builder 서브에이전트 호출
```
context: .experiments/01-designmd-impact/shared/base-prompt.md (only)
output dir: .experiments/01-designmd-impact/v1-no-design/output/
```

### 2. 산출
- `v1-no-design/output/index.html`
- `v1-no-design/output/style.css`

### 3. chatlog 기록
`chatlog/v1-session.md`에 다음을 박제:

```markdown
# V1 Session — DESIGN.md 없이

**Date:** <오늘>
**Inputs:** shared/base-prompt.md
**Subagent:** sc:sc-frontend-architect (ui-builder 역할)

---

## User
> (base-prompt.md 내용 그대로)

## Assistant (ui-builder)
- index.html: <간단 요약 — 어떤 구조로 짰는지 한 단락>
- style.css: <어떤 시스템 토큰 사용했는지 한 단락>
- 자체 평가: <토스 스러움에 대한 LLM의 자기 평가, 어떤 부분이 모자랄 수 있는지>

→ v1-no-design/output/ 작성 완료
```

### 4. 평가 메모 (사람)
일부러 평가 메모를 안 적는다 — V1은 베이스라인이라 raw 결과 그대로.
