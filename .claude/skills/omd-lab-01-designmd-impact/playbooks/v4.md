# V4 Playbook — Iterative Feedback Loop (5 iterations)

## 목적
V3 결과 + 5회의 피드백 turn으로 점진 정제. 피드백을 DESIGN.md의 Preferences 섹션에 누적 → UI 재생성 반복.

## 사전 준비
- `v4-iterative-feedback/feedback-script.md` (5개 항목, 사전 큐레이션)
- 시작점: V3 산출물 복제
  - `v3-describe-to-design/step2-DESIGN.md` → `v4-iterative-feedback/iterations/iter-00/DESIGN.md`
  - `v3-describe-to-design/output/` → `v4-iterative-feedback/iterations/iter-00/output/`

## iter-N 수행 (N = 1..5)

### a. preference-merger 호출
```
input:
  - iter-(N-1)/DESIGN.md
  - feedback-script.md[N]
output:
  - iter-N/DESIGN.md (Preferences 섹션에 N번 피드백 누적, 충돌은 명시)
  - iter-N/design-diff.md (이번 턴 변경분만 추출)
```

### b. ui-builder 호출
```
context: shared/base-prompt.md + iter-N/DESIGN.md + iter-(N-1)/output/ (이전 결과 참고)
output: iter-N/output/
```

### c. chatlog 기록
`chatlog/v4-iter-0N.md`에 박제:
- User: feedback-script.md[N] 본문
- Assistant (preference-merger): preference 누적 결과 + diff
- Assistant (ui-builder): 이번 iter에서 변경된 시각 요소 요약

## 종료
`iter-05` 완료 후 `v4-iterative-feedback/final → iter-05` 심볼릭 링크 (또는 복제).
