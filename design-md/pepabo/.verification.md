## Token architecture (from /inhouse/about/)

Two-tier tokens: **Primitive Tokens** hold direct values (`$blue-600: #1f7ccc`), **Semantic Tokens** reference primitives by intention (`$informative-color: blue 600`). **Flavors** = brand-swappable token collections (Color, Sizing, Icon, Typography, Elevation) layered over a neutral "pepper" base so one system serves multiple brands (minne, SUZURI, lolipop!, ムームードメイン, カラーミーショップ). Five stated principles: enabler-not-enforcer ("a convenient constraint"), living system, neutral prototype, typography-first, environmental honesty.
