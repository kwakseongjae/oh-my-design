### Single-system: Two-tier Freee Blue button + tinted segment cards

freee uses a **height-driven two-tier system** with strict weight-700 typography:
- Header utility: 5px radius / 4×20 / 31-34px / 14px·700
- Hero/page CTA: 8px radius / 10×16-20 / 48px / 15-16px·700
- Two fills: `#2864f0` Freee Blue (Primary) + `#fff` (Outline with `#2864f0` text)
- Segment-tint cards `#ebf3ff` (Freee Blue at ~8% opacity) — never decorative, only for product-segment selectors
- **Weight 700 on every button** — distinctive vs the weight-400 norm of most JP SaaS

### Important conflict — Brand blue HEX

§2 / §11 of the existing DESIGN.md cite **`#285ac8`** as the Freee Blue. Tier 1 live measurement on both surfaces (home header + /accounting hero + segment cards) consistently returns `rgb(40, 100, 240)` = **`#2864f0`** — a meaningfully different (brighter, more saturated) blue.

**Resolution:** `#2864f0` is the **canonical live-CSS value as rendered May 2026**. `#285ac8` may have been the older 2018-era Vibes design-system token, or a brand-style-guide PDF value. The DOM is authoritative for visual rendering today; the docs may have lagged behind a 2024-2025 rebrand. **Both should be considered:** §2 Tokens lists `#285ac8` for legacy compat; live UIs ship `#2864f0`.
