## What is this?

**oh-my-design** lets you pick a real company's design system (Stripe, Vercel, Notion, Linear, and 54 more), customize it through an interactive wizard, and export a production-ready `DESIGN.md` + shadcn/ui CSS variables.

The exported `DESIGN.md` follows the [Google Stitch](https://stitch.withgoogle.com/) format, so any AI coding agent (Claude, Cursor, v0, etc.) can read it and generate pixel-perfect UI that matches your chosen design language.

**No API keys. No AI calls. Everything runs client-side.**

## Features

- **58 Real References** -- Actual design system documents from Stripe, Vercel, Notion, Linear, Figma, Apple, Spotify, Tesla, and more
- **Interactive Wizard** -- A/B style choices for buttons, tables, headers, cards
- **Full Token Control** -- Customize colors, typography, font weight, border radius, dark mode
- **Component Gallery** -- 17 components with live preview (Button, Table, Card, Calendar, Dialog, Toast, Accordion, etc.)
- **shadcn/ui Integration** -- Export ready-to-paste CSS variables for `globals.css`
- **CLI with Config Hash** -- Save your config as a hash, regenerate anywhere
- **Zero AI, Zero Cost** -- All generation is deterministic, client-side, and free
- **No Emojis** -- Enforced icon-only policy with SVG library guidelines

## Quick Start

### Web Builder (Recommended)

```bash
cd web && npm install && npm run dev
```

Open `http://localhost:3333` and start building.

### CLI

```bash
# Interactive mode
node dist/bin/oh-my-design.js

# From a config hash (generated by the web builder)
node dist/bin/oh-my-design.js --config=<YOUR_CONFIG_HASH>
```

### Programmatic API

```typescript
import { loadReference, applyOverrides } from 'oh-my-design';

const ref = loadReference('stripe');
const { designMd, shadcnCss } = applyOverrides(ref, {
  primaryColor: '#6366f1',
  darkMode: true,
}, 'customized');
```

## How It Works

```
1. Pick a Reference    Stripe, Vercel, Notion... (58 companies)
        |
2. Style Taste         A/B choices: sharp vs rounded, minimal vs bordered
        |
3. Design Tokens       Color, font, weight, radius, dark mode
        |
4. Components          Add/remove from 17 component types
        |
5. Export              DESIGN.md + shadcn CSS + JSON tokens + CLI command
```

## Exported DESIGN.md Structure

Follows the [Google Stitch DESIGN.md format](https://stitch.withgoogle.com/docs/design-md/overview/) with extensions:

| Section | Content |
|---------|---------|
| 1. Visual Theme & Atmosphere | Mood, personality, key characteristics |
| 2. Color Palette & Roles | Primary, semantic, surface, shadow colors |
| 3. Typography Rules | Font families, scale, weights, spacing |
| 4. Component Stylings | Buttons, cards, inputs, tables with states |
| 5. Layout Principles | Spacing scale, grid, whitespace |
| 6. Depth & Elevation | Shadow system, surface hierarchy |
| 7. Do's and Don'ts | Design guardrails |
| 8. Responsive Behavior | Breakpoints, touch targets |
| 9. Agent Prompt Guide | Quick reference + example prompts |
| + Iconography & SVG | Icon library, sizing, optimization |
| + shadcn/ui Theme | CSS variables block |
| + Document Policies | No-emoji rule, format compliance |
